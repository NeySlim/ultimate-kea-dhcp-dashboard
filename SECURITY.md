# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in Ultimate Kea DHCP Dashboard, please report it by:

1. **Do NOT** open a public issue
2. Send details to the repository maintainer via GitHub private message
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will respond within 48 hours and work on a fix promptly.

## Security Best Practices

When deploying this dashboard:

- Always use SSL/TLS encryption
- Restrict access using firewall rules
- Keep configuration files (containing paths) readable only by root
- Regularly update dependencies
- Run on a dedicated network segment if possible
- Monitor access logs regularly
